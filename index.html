SoundState

SoundState is a smart playlist generator that allows a user to filter their Spotify favourites according to their mood. It integrates AI to create a unique image based on each playlist, which can then be shared. We built this as a team of 4 for our final 2 week project at bootcamp.

TECH

  • Ruby on Rails
  • PostgreSQL
  • Spotify API
  • OpenAI API
  • Sass
  • Heroku

DEMO

  • Over Here >>
  • NB. This app is currently in development phase. username: soundstatedemo@gmail.com; password: SoundStateDemo1160

Why This Project?

Music is one of the things I enjoy the most. I wanted finer control over my Spotify library without the pain of having to manually create one, so I pitched this idea as the final project. I feel music is deeply personal - what we listen to depends so much on our mood or the occasion; yet music also has this great power to bring people together. SoundState is unique in that it integrates the OpenAI API to generate an image based on the mood/songs of each playlist, in order to make sharing and discovering music more visual and fun.

I also felt it would be a great learning opportunity - integrating the Spotify and OpenAI APIs, becoming more proficient at reading documentation, and developing my front-end skills.

Learning Points

I was team lead for this project and got to hone management as well as technical skills. Building the app was a huge learning experience. Some of the highlights:

  • Mobile-First Design and how to implement Progressive Web App features
  • Defining a sound database schema, ensuring validations to ensure data integrity and consistency
  • Deepening understanding of MVC architecture
  • Applying RESTful principles
  • Keeping code clean and maintainable through the use of components, partials, service objects
  • Integrating Spotify and OpenAI APIs
  • Working with OAuth
  • Figma prototyping, collaboration via Github, deployment through Heroku, image hosting on on Cloudinary
  • Project management & Agile methodologies - Kanban workflow, daily stand-up meetings. We had to be flexible and adaptable when we realised some API limitations. For example, using embedded iframes instead of web SDK.

Check out the Github README for more details.

No Matter How Small

No Matter How Small is a medical app for paediatric anaesthetists working in Belfast. It automates drug dose calculations and makes it easy to access relevant guidelines and protocols on the go.

TECH

  • Vanilla JS
  • Express JS
  • Node JS
  • Bootstrap
  • Northflank

DEMO

  • Over Here >>

Why This Project?

I was returning to work as a doctor in children's anaesthesia, a sleep-deprived first-time mum worried about making a drug calculation error. I decided to build this app to automate calculations (instead of relying on mental arithmetic), and to allow quick referencing of care guidelines. This was also aimed at improving the onboarding process of anaesthetic registrars rotating departments every 3 months.

On a personal level, this project is close to my heart because it is what sparked my interest in coding. The cherry on top was when the app got adopted at work! I experienced first hand what a great skill this is to have.

Learning Points

As I was learning on my own, one of by biggest challenges was knowing what best practice is and how to structure my code. I switched from watching YouTube videos to Colt Steele's Udemy course on web development to try to address this. Some learning highlights:

  • Vanilla JS: working with objects, arrays, loops and iteration
  • Responsive web design (it was really important for this to be good on mobile devices to be used on the go)
  • How to customise colours in Bootstrap
  • Routing in Express JS
  • DOM manipulation
  • Rendering PDFs in iframes
  • HTML tables
  • Working with partials to keep code maintainable

Ch-Eye-Nese

Ch-Eye-Nese is a fun audiovisual guide to help eye doctors communicate with their patients in Chinese. It translates the most common phrases used in an ophthalmology clinic.

TECH

  • HTML/CSS
  • JS

DEMO

  • Over Here >>

Why This Project?

I built this for foreign doctors working in Singapore (my home country) who might find it difficult to converse with elderly Chinese patients in clinic. Clicking on each image in mobile / hovering on desktop overlays the phonetic pronunciation of each phrase and plays an audio clip as well.

Learning Points

  • CSS grid and overlays
  • Using the audio tag in HTML and playing audio files with JS
